# 产品设计

> 与 `docs/PRD.md` 同步：学生 / 教师 / 管理员角色、注册与学科审核范围（**§14 决策已确认**）

## 1. 目标用户

| 角色 | 诉求 |
|------|------|
| 学生 | 注册学习账号；生成/提交作品；在广场学习公开课；查看审核结果 |
| 教师 | 注册并选择授课学科；生成教学媒体；**仅审核本学科学生作品** |
| 管理员 | 最高权限：全站审核、用户治理、系统配置、任务运维 |

## 1.1 已确认决策（摘自 PRD §14）

1. 旧 `user` → 迁移为 `student`
2. 教师作品 → 仅管理员审核
3. 无对口教师 → 学生待审进管理员队列
4. 教师注册 → 首期不要邀请码
5. 教师可自改授课学科（≥1）
6. 审核越权详情 → 统一 `404`
7. 学生可用全部生成档位

## 2. 核心用户故事

1. **注册**  
   作为新用户，我选择「学生」或「教师」完成注册；若选教师，必须勾选至少一个授课学科（如物理）。
2. **生成**  
   作为学生/教师，我选择「初中 / 物理 / 机械能与能量」，输入「能量守恒定律」，系统生成 1–2 分钟配音教学视频。
3. **学生送审**  
   生成成功后，学生完善标题摘要并提交审核。
4. **学科教师审核**  
   物理老师在「待我审核」中看到该课；数学老师看不到。老师可选择通过或驳回并填写意见。
5. **管理员兜底与治理**  
   管理员可审核任意学科/任意角色作品；无对口教师时学生待审进入管理员队列；可禁用用户、调整教师学科与系统配置。
6. **分享学习**  
   审核通过后，课程进入公开库；学生可按「物理-初二」筛选观看，并可复制再创作。

## 3. 信息架构

### 用户端（按角色裁剪菜单）

- 登录 / 注册（角色 + 教师学科）
- 首页：热门课程、学科入口
- 创建课程视频：表单（学科/年级/知识点/补充说明）
- 我的任务：排队/生成中/成功/失败
- 我的作品：私有草稿 + 待审 + 已发布/驳回
- 课程广场：按分类浏览/搜索/播放
- 个人中心：资料、密码；教师含授课学科；**模型设置（TTS/文生图/文生视频）**

### 教师增量

- 待我审核（本学科学生 pending）
- 审核记录

### 管理端

- 仪表盘：任务量、成功率、各学科待审积压
- 用户管理（角色/状态/教师学科）
- 全站课程审核
- 分类管理（学科树）
- 任务运维（重试/取消）
- 技能与模型配置（provider、并发、超时、审核策略）

## 4. 课程输入模型

```json
{
  "subject": "physics",
  "grade": "grade8",
  "chapter": "机械能与能量",
  "topic": "能量守恒定律",
  "learningGoals": ["理解能量守恒", "区分机械能守恒与能量守恒"],
  "difficulty": "standard",
  "durationHint": "90s",
  "styleNotes": "适合暑假复习，口语化",
  "outputProfile": "teaching_video_full"
}
```

`outputProfile` 决定 skill 串法：

| profile | 产物 |
|---------|------|
| `infographic_only` | 9:16 信息图 |
| `teaching_video_full` | 分镜 + TTS + 1080p 配音视频 |
| `package_all` | 信息图 + 概念图 + 视频 + 封面 |

默认推荐：`teaching_video_full`；需要信息图、概念图和封面时选择 `package_all`。

## 5. 生成结果验收标准（对齐实测）

参考已验证产物 `energy-conservation.mp4`：

- 时长约 60–120 秒（7 段左右）
- 1920×1080 H.264 + AAC
- 含字幕/章节动画
- 可附封面 3:4、信息图 9:16
- 元数据记录 skill 版本、provider、耗时
