# AI 教学媒体平台详细设计

## 1. 任务输入协议

```json
{
  "outputProfile": "package_all",
  "subject": "physics",
  "grade": "grade8",
  "chapter": "机械能与能量",
  "topic": "能量守恒定律",
  "learningGoals": ["理解能量守恒"],
  "style": "cozy-handdrawn",
  "article": "可选 Markdown 原文",
  "referenceImages": [],
  "autoCreateCourse": true
}
```

文章型档位使用 `article`，知识点型档位使用 `topic`。服务端限制文本长度，拒绝空输入和未知 profile。

## 2. Pipeline Adapter

### 教学视频

`buildStoryboard` → `edu-teaching-animation/scripts/minimax_tts.py` → `scaffold_video.py` → `npx hyperframes render`。

### 信息图/封面/技术插图

先写 manifest 和 prompt 文件，再调用 `ai-image-generator/scripts/generate.py`。比例固定为信息图 `9:16`、封面 `3:4`、文章插图 `16:9`。每次调用的 PNG、TXT、JSON 都被扫描并登记为资产。

### 文章解说视频

将文章转换为章节 storyboard，调用 `article-explainer-video/scripts/tts_pipeline.py` 与 `scaffold.py`，随后使用 HyperFrames 渲染。文章插图可在配置了图片 Key 时生成；缺 Key 时任务明确失败，不使用伪图片冒充交付。

## 3. 状态与幂等

- Worker 只处理 `queued`；抢占后立即写 `running`。
- 同一 Job 的工作目录固定，脚本支持 `--skip-existing` 的地方复用音频。
- 重试会清除错误和完成时间，但保留输入快照与历史工作目录。
- 资产以 `(job_id, type, path)` 作为应用层去重键。

## 4. 安全与可观测性

- 所有私有 API 使用 Bearer session；课程和 Job 按 user_id 做授权检查。
- 子进程参数使用数组传递，不拼接用户输入为 shell 命令。
- 记录阶段、耗时、命令退出码和 stderr 摘要；不记录 API Key。
- 生产环境应增加限流、内容安全审核、对象存储签名 URL 和日志脱敏。

## 学科与知识点目录设计

### 选择器

生成页 `KnowledgeSelector`：

1. 选学科/年级 → 拉 `GET /catalog/categories` 与 `knowledge-points`
2. 关键字搜索 → `q` 匹配 topic/chapter/summary/keywords
3. 选中后回填 `chapter`、`topic`，并尽量带上 `learningGoals`

### 管理端

- 独立路由视图 `admin-knowledge`
- 左：筛选列表；右：sticky 编辑表单
- 同步接口按 `pack_key` 幂等 upsert

### 与 Pipeline

- `animation_pack` / topic 关键词参与 `detectFamily` 与 SVG 零件装配
- 知识驱动 storyboard 优先使用摘要与学习目标，避免空泛旁白

详见 `docs/knowledge-catalog.md`、`docs/api.md`。
