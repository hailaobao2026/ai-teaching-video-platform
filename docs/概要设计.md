# AI 教学媒体平台概要设计

## 1. 技术架构

```text
React 19 + Vite
        │ REST JSON / Bearer Session
Express API (:3002)
   ├── Auth / Catalog / Jobs / Courses / Admin / Assets
   ├── MySQL 8 或 memory-db.json
   └── uploads 静态产物
        │ queued job
独立 Node Worker（默认并发 1）
        │ child_process
ai-teaching-media skill
   ├── storyboard / infographic / diagram
   ├── Edge 或 Minimax TTS
   ├── scaffold / tts_pipeline
   ├── HyperFrames + ffmpeg
   └── ai-image-generator 四 Provider
```

## 2. 模块职责

| 模块 | 职责 |
|---|---|
| Web Client | 表单、任务列表、课程广场、审核页面、学科知识点管理和视频播放 |
| API Server | 参数校验、会话鉴权、任务入队、课程审核、学科知识点目录、资产读取 |
| Repository | 屏蔽内存/MySQL 差异，提供 Job/Course/Asset 持久化 |
| Pipeline Adapter | 将统一任务输入转换成具体 skill CLI，收集产物 |
| Worker | 抢占队列、阶段进度、异常捕获、课程草稿创建 |
| Skill | 内容理解、提示词、TTS、生图、HTML 视频渲染的实际执行层 |

## 3. 运行时流程

1. API 校验请求并创建 `generation_jobs`。
2. Worker 以事务或内存锁抢占最早的 queued 任务。
3. Adapter 在 `ARTIFACTS_ROOT/<jobId>` 创建隔离目录并写输入快照。
4. 按输出档位执行脚本，阶段进度写回数据库。
5. 将 MP4/PNG/JSON/TXT 复制到 uploads，创建 `course_assets`。
6. 成功后更新 Job 并按配置创建课程草稿；失败保留工作目录和错误。

## 4. 部署

- 开发：API、Worker、Vite 三进程，`USE_MYSQL=false`。
- 集成：Docker Compose 启动 MySQL、API、Worker、Nginx。
- 生产演进：对象存储替换 uploads，Redis/BullMQ 替换 MySQL 抢占，Worker 独立扩容。

## 用户模型设置

学生/教师/管理员可配置 TTS、文生图、教学视频渲染偏好；优先级：任务 > 用户设置 > 系统配置 > 环境变量。详见 `docs/user-model-settings.md`。


## 学科与知识点目录

- 表：`subjects`、`knowledge_points`
- 公开 API：catalog subjects/grades/categories/knowledge-points
- 管理 API：CRUD + 多来源同步（动画包/ChemAIForge/初中语英史地政）
- 文档：`docs/knowledge-catalog.md`
